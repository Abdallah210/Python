Ensembles et dictionnaires¶

20-01-2021

Objectifs

  • Découvrir deux nouvelles structures de données : ensembles et dictionnaires.

Structures itérables¶

En Python, une structure de données struct est dite itérable si on peut écrire une instruction de la forme

for a in struct:
    ...

Toutes les structures séquentielles sont itérables. Mais il existe des structures itérables qui ne sont pas séquentielles. Nous allons déécouvrir deux telles structures qu’offre le langage Python :

  • les ensembles
  • et les dictionnaires.

Ensembles¶

Comme les listes, les ensembles sont des structures de données permettant de regrouper plusieurs valeurs en une seule. Mais à la différence des listes,

  • les ensembles ne peuvent pas avoir plusieurs occurrences d’une même valeur,
  • les éléments d’un ensemble ne sont pas rangés dans un ordre déterminé permettant l’usage d’indices.

Constructions¶

Construction littérale¶

La plus simple des constructions d’un ensemble consiste à placer entre deux accolades l’énumération de ses éléments

>>> {"timoleon", 1, 2.5}
{1, 2.5, 'timoleon'}
>>> {1, 2 - 1, -1 + 2}
{1}

Comme le montre le second exemple ci-dessus, les ensembles n’acceptent pas la répétition d’un même élément. Et le premier exemple montre que l’ordre des éléments dans une énumération ne se retrouve pas dans la réponse fournie par l’interpréteur.

Construction avec set¶

Le type des ensembles se nomme set. La fonction set permet la construction d’un ensemble à partir de n’importe quelle valeur itérable. En voici quelques exemples :

>>> set('timoleon')
{'e', 'i', 'l', 'm', 'n', 'o', 't'}
>>> set([3, 1, 4, 1, 5, 9, 2])
{1, 2, 3, 4, 5, 9}
>>> set(range(1, 10, 2))
{1, 3, 5, 7, 9}

L’ensemble vide se construit avec la fonction set sans argument

>>> set()
{}

Avertissement

À la différence des chaînes, tuples et listes, l’ensemble vide ne peut pas être écrit littéralement par l’expression {} car on obtient un dictionnaire (voir plus bas) et non un ensemble :

>>> faux_ensemble = {}
>>> type(faux_ensemble)
<class 'dict'>

Il faut impérativement utiliser la fonction set pour construire un ensemble vide :

>>> vrai_ensemble = set()
>>> type(vrai_ensemble)
<class 'set'>

Construction en compréhension¶

Enfin, comme pour les listes, il est possible de définir un ensemble en compréhension. Voici par exemple l’ensemble des carrés des entiers compris entre -2 et 5.

>>> set(k*k for k in range(-2,6))
{0, 1, 4, 9, 16, 25}

Remarque sur le contenu des ensembles¶

Avertissement

À la différence des listes, un ensemble ne peut pas contenir tout type de données.

Comme le montre l’exemple ci-dessous, on ne peut pas mettre une liste dans un ensemble.

>>> {[1, 2, 3]}
...
TypeError: unhashable type: 'list'

De même on ne peut pas mettre d’ensemble dans un ensemble.

>>> {{1, 2, 3}}
...
TypeError: unhashable type: 'set'

Quels sont les types de données qui peuvent figurer dans un ensemble ? Ce sont tous les types dits hachables. Nous ne préciserons pas ce qu’est un type hachable. Nous nous contenterons de dire que ce sont tous les types simples (nombres entiers ou flottants, booléens) ou correspondant à des structures non mutables (chaînes, intervalles, tuples).

Nombre d’éléments d’un ensemble¶

Le nombre d’éléments d’un ensemble est donné par la fonction len.

>>> len(set('timoleon'))
7

Parcours des éléments d’un ensemble¶

Avertissement

L’absence de contrôle de l’ordre des éléments enlève l’intérêt de la notation indicielle.

>>> ens = {1,2,3}
>>> ens[0]
TypeError: 'set' object does not support indexing

En revanche l’opérateur in permet de tester l’appartenance d’une valeur à un ensemble.

>>> 3 in ens
True
>>> 5 in ens
False

Les ensembles sont des structures de données itérables.

>>> for c in ens:
       print (c, end=', ')
1, 2, 3,

Méthodes sur les ensembles¶

Les méthodes index et count ne sont pas des méthodes définies pour les ensembles. La première parce que la notion d’indice n’a aucun sens pour les éléments d’un ensemble. La seconde parce que dans un ensemble une valeur est soit absente, soit présente une seule fois, et la méthode count perd tout intérêt.

  1. Méthodes add , remove

    Ces méthodes ajoutent et retirent un élément dans un ensemble. Elles modifient l’ensemble et ne renvoient aucune valeur (None).

    >>> ens1 = {1, 2, 3}
>>> 4 in ens1
False
>>> ens1.add(4)
>>> 4 in ens1
True
>>> ens1.remove(4)
>>> 4 in ens1
False

    Avertissement

    La méthode remove appliquée à un élément n’appartenant pas à un ensemble déclenche l’exception KeyError.

    >>> {1, 2, 3}.remove(4)
...
KeyError: 4

    Retenons donc que les ensembles sont des structures de données mutables.

  2. Méthodes union, intersection

    Ces méthodes renvoient la réunion et l’intersection de l’ensemble passé en paramètre et de celui sur lequel elles s’appliquent, et qu’elles ne modifient pas.

    >>> ens1.union({3, 4, 5})
{1, 2, 3, 4, 5}
>>> ens1
{1, 2, 3}
>>> ens1.intersection({3, 4, 5})
{3}
>>> ens1
{1, 2, 3}

  3. Méthodes issubset, isdisjoint

    La méthode issubset est un prédicat qui renvoie True si l’ensemble sur lequel elle s’applique est un sous-ensemble de celui passé en paramètre, et False dans le cas contraire.

    >>> ens1.isubset({1, 3})
False
>>> ens1.isubset({1, 3, 2, 4})
True

    La méthode isdisjoint est aussi un prédicat pour décider si deux ensembles sont disjoints.

    >>> ens1.isdisjoint({1, 3})
False
>>> ens1.isdisjoint({4, 5, 6})
True

  4. Méthode copy

    Méthode analogue à celle pour les listes, et utile puisque les ensembles sont mutables.

Les dictionnaires¶

Les dictionnaires sont des structures de données permettant d’associer une valeur à une clé.

Ainsi une liste de couples comme l = [('banane', 2.59), ('pomme',1.95), ('orange',1.95)] qui associe un prix au kg à des fruits, peut être considérée comme une liste d’association dans laquelle les fruits sont les clés et les prix les valeurs associées.

Dans un dictionnaire, il ne peut y avoir qu’une seule association pour une clé donnée. Dit autrement, dans un dictionnaire les clés sont uniques.

Constructions¶

Construction littérale¶

Les constructions littérales de dictionnaires sont obtenues par énumération explicite des associations clé-valeur sous la forme cle : valeur, associations séparées par des virgules, le tout entre accolades.

>>> tarifs = {'banane':2.59, 'pomme':1.95, 'orange':1.95}
>>> tarifs
{'banane': 2.59, 'orange': 1.95, 'pomme': 1.95}
>>> type(tarifs)
<class 'dict'>

Construction avec la fonction dict¶

Une autre construction possible du même dictionnaire utilise la fonction dict.

>>> tarifs = dict([('banane', 2.59), ('pomme',1.95), ('orange',1.95)])

Avertissement

toute valeur ne peut pas servir de clé. Seules les valeurs non mutables peuvent servir de clé dans une association.

Par exemple, la construction suivante échoue car on tente de construire un dictionnaire avec une association dont la clé est une liste qui est une structure mutable.

>>> {[] : 1}
Traceback (most recent call last):
  File "<pyshell#11>", line 1, in <module>
    {[] : 1}
TypeError: unhashable type: 'list'

Dictionnaire vide¶

Un dictionnaire qui ne contient aucune association est un dictionnaire vide.

Pour obtenir un dictionnaire vide, on peut donner l’expression

>>> {}
{}

ou bien

>>> dict()
{}

Taille d’un dictionnaire¶

La taille d’un dictionnaire, c’est-à-dire le nombre d’associations qu’il contient, est donnée par la fonction len.

>>> len(tarifs)
3

Consultation¶

Comme les ensembles, les dictionnaires ne sont pas des structures séquentielles, et leurs éléments ne sont donc pas indexés par des nombres entiers.

En revanche, on accède à la valeur associée à une clé k dans un dictionnaire d grâce à l’expression d[k].

>>> tarifs['banane']
2.59
>>> tarifs['orange']
1.95

Toute tentative d’accéder à la valeur associée à une clé qui n’existe pas se solde par une exception KeyError.

>>> tarifs['fraise']
Traceback (most recent call last):
  File "<pyshell#13>", line 1, in <module>
    tarifs['fraise']
KeyError: 'fraise'

Existence d’une association pour une clé donnée¶

On vient de voir que l’accès à la valeur associée à une clé n’existant pas déclenche une exception. Il peut donc être utile de vérifier l’existence de la clé dans un dictionnaire. C’est l’opérateur in qui permet cette vérification.

>>> 'banane' in tarifs
True
>>> 'fraise' in tarifs
False

Modification¶

Les dictionnaires sont des structures mutables. On peut modifier la valeur associée à une clé.

Par exemple le prix des bananes baisse.

>>> tarifs['banane'] = 2.30
>>> tarifs
{('banane', 2.3), ('orange',1.95), ('pomme',1.95)}

Ajout d’une association¶

On peut aussi modifier un dictionnaire en ajoutant une association.

Les fraises arrivent (et sont cheres).

>>> tarifs['fraise'] = 7.5
>>> tarifs
{'banane': 2.3, 'orange': 1.95, 'pomme': 1.95, 'fraise': 7.5}

Suppression d’une association¶

On peut enfin supprimer une association dans un dictionnaire. On le fait avec l’instruction del.

>>> del tarifs['fraise']
>>> tarifs
{'banane': 2.3, 'orange': 1.95, 'pomme': 1.95}

Itération¶

Tout commme les listes, les chaînes de caractères, les intervalles et les ensembles, les dictionnaires sont des structures itérables. Il est donc très facile d’effectuer un traitement sur toutes les associations d’un dictionnaire à l’aide d’une boucle pour.

Voici une itération pour imprimer toutes les clés d’un dictionnaire :

>>> for k in tarifs:
       print(k)

banane
orange
pomme

et une autre pour imprimer toutes les valeurs associées :

>>> for k in tarifs:
       print (tarifs[k])

2.3
1.95
1.95

Avertissement

Les dictionnaires ne sont pas des structures séquentielles. Il n’est donc pas possible dans un traitement itératif des données contenues dans un dictionnaire de s’appuyer sur un quelconque ordre de ces données.

Quelques applications¶

Représentation de dates¶

Une date peut être représentée de différentes façons. L’une d’elles est la donnée de trois nombres entiers donnant le jour, le mois et l’année. Ces trois nombres peuvent être rassemblés dans une liste ou un tuple. Mais y a-t-il un ordre canonique dans lequel ranger ces trois nombres dans une liste ou un tuple ? Et une fois cet ordre choisi comment interpréter une liste ou un tuple de trois entiers ? Par exemple, le tuple (1,2,3) représente-t-il le 1er février de l’an 3, ou le 3 février de l’an 1, ou encore l’une des quatre autres interprétations possibles ?

L’utilisation de dictionnaire permet de nommer les valeurs avec des clés significatives enlevant toute ambiguïté. Pour représenter la date d’aujourd’hui, on peut construire un dictionnaire à trois associations de clés 'jour', 'mois' et 'annee'.

>>> aujourdhui = {'jour': 25, 'mois': 2, 'annee': 2016}

Une fois choisie une telle représentation des dates, on peut réaliser des fonctions pour obtenir diverses opérations sur les dates. Par exemple une fonction qui calcule la date du lendemain.

>>> lendemain(aujourdhui)
{'mois': 1, 'jour': 26, 'annee': 2016}

Tabulation d’une fonction¶

Les dictionnaires peuvent être utilisés pour tabuler des fonctions.

>>> carres = dict([(k, k*k) for k in range(100)])

On peut désormais utiliser le dictionnaire carres pour obtenir le carré de n’importe quel nombre entier compris entre 0 et 99 inclus, sans avoir à calculer ce carré.

Compter¶

Voici comment il est possible de compter le nombre d’occurrences de chacun des caractères contenus dans une chaîne à l’aide d’un dictionnaire.

>>> s = "la gardienne du campus dessine"
>>> d = dict()
>>> for c in s:
       if c in d:
          d[c] += 1
       else:
          d[c] = 1

>>> for c in d:
       print(c,':', d[c])

a : 3
i : 2
l : 1
d : 3
g : 1
c : 1
n : 3
r : 1
  : 4
s : 3
e : 4
u : 2
p : 1
m : 1
