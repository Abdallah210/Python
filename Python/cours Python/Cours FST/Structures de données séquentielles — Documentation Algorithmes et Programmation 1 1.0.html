Retour sur les chaînes, tuples et intervalles¶

Les chaînes de caractères (str), les tuples (tuples) et les intervalles (range) sont des structures de données séquentielles déjà rencontrées lors du premier semestre.

>>> s = "timoleon"
>>> t = (1, 2, 3)
>>> r = range(1,10,2)

Ces trois structures de données partagent un certain nombre de points communs.

  1. Elles ont toutes une longueur, déterminée par le nombre d’éléments contenus dans la structure. C’est la fonction len qui donne la longueur d’une valeur de l’un de ces types.

    >>> len(s)
8
>>> len(t)
3
>>> len(r)
5

  2. Les éléments de ces structures peuvent être désignés par un indice. Cet indice est un nombre entier compris entre 0 et la longueur moins un.

    >>> s[0]
't'
>>> s[len(s) - 1]
'n'
>>> t[0]
1
>>> t[len(l) - 1]
3
>>> r[0]
1
>>> r[len(r) - 1]
9

  3. L’opérateur in permet de tester l’appartenance d’une valeur dans chacune de ces structures.

    >>> 't' in s
True
>>> 'u' in s
False
>>> 2 in t
True
>>> 4 in t
False
>>> 1 in r
True
>>> 2 in r
False

  4. On peut effectuer des traitements itératifs sur ces structures de données à l’aide de la boucle for. C’est pourquoi on dit de ces structures qu’elles sont itérables.

    >>> for c in s:
       print(c, end=', ')
t, i, m, o, l, e, o, n,
>>> for c in t:
       print(c, end=', ')
1, 2, 3,
>>> for c in r:
       print(c, end=', ')
1, 3, 5, 7, 9,

Mais elles sont aussi différentes.

  1. Seuls les tuples peuvent contenir des éléments de types hétérogènes. Les chaînes de caractères ne contiennent que des caractères, et les intervalles que des nombres entiers.

    Voici par exemple un tuple contenant des éléments de différents types.

    >>> (s, l, r)
('timoleon', [1, 2, 3], range(1, 10, 2))

  2. Seuls les chaînes et les tuples peuvent être concaténées avec l’opérateur +. La concaténation n’a pas de sens pour les intervalles.

    >>> 'Bonjour ' + s
'Bonjour timoleon'
>>> t + t
(1, -3, 3, 1, -3, 3)
    >>> r + r
    ...
TypeError: unsupported operand type(s) for +: 'range' and 'range'

Méthodes¶

La notion de méthode provient de la programmation orientée objets.

Les méthodes sont des sortes de fonctions qui sont attachées aux objets. Syntaxiquement, lorsqu’on veut faire appel à une méthode m attachée à un objet o, on utilise une notation pointée et on écrit

o.m(x1, x2, ...)

alors qu’avec une fonction on aurait écrit

m(o, x1, x2, ...)

Méthodes communes aux chaînes, tuples et intervalles¶

  1. Méthode index

    Cette méthode permet d’obtenir le plus petit indice où l’on peut trouver l’élément passé en paramètre dans l’objet sur lequel elle s’applique. Cette méthode déclenche une exception ValueError si l’élément cherché n’est pas présent.

    >>> "timoleon".index("o")
3
>>> "timoleon".index("z")
    ...
ValueError: substring not found
>>> range(1,10,2).index(9)
4

  2. Méthode count

    Cette méthode permet d’obtenir le nombre d’occurences du paramètre (C’est-à-dire le nombre de fois où le paramètre apparaît dans l’objet).

    >>> "timoleon".count("o")
2
>>> "timoleon".count("z")
0
>>> range(1,10,2).count(9)
1

Ces deux méthodes sont les deux seules que possèdent les intervalles et tuples. En revanche les chaînes en ont beaucoup d’autres.

Méthodes spécifiques aux chaînes¶

Les chaînes de caractères possèdent de très nombreuses méthodes qui leur sont propres. Il n’est pas question de les passer toutes en revue ici.

  1. Méthodes isupper, islower, upper, lower

    La méthode isupper est un prédicat sans paramètres, qui renvoie True, si la chaîne sur laquelle elle s’applique contient au moins une lettre, et si parmi ces lettres il n’y a que des capitales, et renvoie False dans le cas contraire. La méthode islower est analogue pour les bas de casses.

    >>> "123".isupper()
False
>>> "TOTO1".isupper()
True
>>> "ToTO".isupper()
False

    La méthode upper n’a pas de paramètre, et renvoie une copie de la chaîne sur laquelle elle s’applique où les lettres sont remplacées par les lettres capitales correspondantes. La méthode lower est analogue pour la transformation des lettres en bas de casse.

    >>> "Timoleon".upper()
"TIMOLEON"
>>> "Timo12".upper()
"TIMO12"

  2. Méthodes rstrip, lstrip, strip

    La méthode rstrip, utilisée sans paramètre, renvoie une copie de la chaîne à laquelle elle s’applique en supprimant tous les caractères d’espacement situés à la fin.

    >>> 'Timoleon est un homme politique grec     '.rstrip()
'Timoleon est un homme politique grec'
>>> 'Timoleon est un homme politique grec'.rstrip()
'Timoleon est un homme politique grec'

    Utilisée avec un paramètre qui est une chaîne de caractères, ce sont tous les caractères de ce paramètre situés à droite qui sont supprimés de la copie.

    >>> 'Timoleon est un homme politique grec'.rstrip('cegrz')
'Timoleon est un homme politique '
>>> 'Timoleon est un homme politique grec'.rstrip('z')
'Timoleon est un homme politique grec'

    Les méthodes lstrip et strip sont analogues à rstrip mais pour la gauche de la chaîne (lstrip) et pour les deux côtés (strip).

  3. Méthodes split et join

Les listes¶

Une liste est une structure de données séquentielle permettant de regrouper en un seul objet plusieurs valeurs pouvant être de types différents. En ce sens, les listes sont semblables aux tuples. Nous verrons néanmoins un peu plus loin en quoi elles diffèrent des tuples.

Constructions¶

Nous allons voir plusieurs façons de construire des listes.

Construction littérale¶

De façon semblable aux tuples, on construit une liste littérale en énumérant ses éléments séparés par des virgules, le tout entre crochets.

>>> [1, 'a']
[1, 'a']

Une liste peut être vide

>>> []
[]

Les listes peuvent contenir toute sorte d’objets y compris des listes

>>> [(1, 2), ['a', 'b']]
[(1, 2), ['a', 'b']]

Construction avec list¶

La fonction list est un autre moyen de construire des listes. Elle s’emploie avec zéro ou un argument, et dans ce cas l’argument doit être un itérable.

>>> list()
[]
>>> list('timoleon')
['t', 'i', 'm', 'o', 'l', 'e', 'o', 'n']
>>> list(range(5))
[0, 1, 2, 3, 4]

La fonction list peut même s’appliquer à une liste.

>>> l1 = list('timoleon')
>>> l2 = list(l1)
>>> l2
['t', 'i', 'm', 'o', 'l', 'e', 'o', 'n']

Avertissement

On peut penser que, dans l’exemple précédent, au lieu d’écrire l’instruction l2 = list(l1) on pourrait écrire l2 = l1. Mais si en apparence cela semble être équivalent, en réalité cela ne l’est pas, comme on peut le voir dans la session ci-dessous :


On voit qu’en mémoire coexistent deux listes identiques en longueur et en contenu mais différentes en localisation. Les variables l1 et l2 pointent chacune vers une de ces deux listes. En revanche la variable l3 pointe vers la même liste que l1.

On verra que cela peut poser des problèmes à cause de la mutabilité des listes.

Construction par concaténation¶

Les listes peuvent être concaténées entre elles avec l’opérateur +.

>>> [3, 1, 4] + [1, 5, 9, 2]
[3, 1, 4, 1, 5, 9, 2]

Cette opération itérée permet de construire des listes aussi longues qu’on veut. Voici par exemple comment obtenir la liste des caractères d’une chaîne sans utiliser la fonction list.

>>> liste_car = []
>>> for c in 'timoleon':
        liste_car = list_car + [c]
>>> liste_car
['t', 'i', 'm', 'o', 'l', 'e', 'o', 'n']

Avertissement

Dans cet exemple, il faut bien prendre garde de ne pas écrire list_car = list_car + c. En effet list_car est une liste et c est une chaîne et Python ne sait pas concaténer une liste et une chaîne.

>>> ['t', 'i'] + 'm'
...
TypeError: can only concatenate list (not "str") to list

Construction en compréhension¶

La construction en compréhension est un moyen puissant, élégant et concis, que le langage Python permet pour les listes et les tuples.

La syntaxe pour cette construction dans sa forme la plus simple est [<expression> for <var> in <iterable>].

Voici par exemple la liste des carrés des entiers de 0 à 9

>>> [k**2 for k in range(10)]
[0, 1, 4, 9, 16, 25, 36, 49, 64, 81]

On peut combiner plusieurs itérations. Par exemple pour obtenir la liste de tous les couples d’entiers compris entre 0 et 2

>>> [(x, y) for x in range(3) for y in range(3)]
[(0, 0), (0, 1), (0, 2), (1, 0), (1, 1), (1, 2), (2, 0), (2, 1), (2, 2)]

On peut aussi filtrer les éléments qu’on veut avec une condition. Voici comment obtenir la liste des voyelles d’une chaine

>>> [c for c in 'timoleon' if c in 'aeiou']
['i', 'o', 'e', 'o']

Longueur d’une liste¶

La longueur d’une liste est le nombre d’éléments de cette liste. Comme pour les autres structures, c’est la fonction len qui donne la longueur d’un tuple.

>>> len(list('timoleon'))
8
>>> len([(1, 2), ['a', 'b']])
2

Accès aux composantes¶

Comme les chaînes, les tuples et les intervalles, les listes sont des structures de données séquentielles. Les éléments des listes sont donc indexés de 0 à n−1n−1 si nn est la longueur de la liste.

>>> l = list('timoleon')
>>> l[0]
't'
>>> l[7]
'n'
>>> l[8]
...
IndexError: tuple index out of range

Égalité de listes¶

Deux listes sont égales si elles ont les mêmes éléments dans le même ordre. En particulier deux listes égales ont même longueur.

L’égalité de deux listes peut être testée avec l’opérateur usuel de comparaison en Python ==.

>>> l1 = list('timoleon')
>>> l2 = list(l1)
>>> l1 == l2
True

Comme déjà signalé plus haut deux listes peuvent être égales sans être situées au même endroit en mémoire. C’est le cas des deux listes de l’exemple ci-dessus.

Si on veut savoir si deux variables pointent vers la même liste on peut utiliser l’opérateur is.

>>> l2 is l1
False
>>> l3 = l1
l3 is l1
True

Opérateur in¶

L’appartenance d’une valeur à une liste peut être testée à l’aide de l’opérateur in :

>>> 't' in l
True
>>> 'u' in l
False

Itération¶

Comme les autres structures séquentielles, on peut itérer sur les éléments d’une liste

>>> for x in ['a', 'b', 'c']:
        print(x, end='')
abcd

Si le traitement itératif nécessite l’usage des indices des éléments de la liste, alors on itère sur les indices.

Mutablilité¶

Voici ce qui en Python distingue les listes des tuples (et aussi des chaines et intervalles) : la mutabilité.

Dire qu’en Python les listes sont mutables signifie qu’il est possible de modifier la longueur et le contenu des listes sans avoir à les reconstruire.

On peut modifier un élément d’une liste l par une instruction d’affectation de la forme l[i] = <expression>. Le membre gauche de cette affectation (l[i]) désigne la position d’indice i de la liste l. Cette instruction a pour effet de changer la valeur de l’élément d’indice i de l par la valeur de <expression>. Par exemple:

>>> l1 = list('timuleon')
>>> l1
['t', 'i', 'm', 'u', 'l', 'e', 'o', 'n']
>>> l1[3] = 'o'
>>> l1
['t', 'i', 'm', 'o', 'l', 'e', 'o', 'n']

Pour être valide, l’indice utilisé dans une telle instruction doit être un indice dans un intervalle acceptable. En particulier, il n’est pas possible par ce moyen d’augmenter la longueur d’une liste.

>>> l1[8] = 'z'
...
IndexError: list assignment index out of range

Avertissement

Partage de valeur (ou phénomène d’aliasing)

Dans la session ci-dessous, trois variables sont définies : l1 et l3 sont deux listes physiquement égales, tandis que l2 est initialement une copie de l1.

Une modification de l’élément d’indice 3 dans la liste pointée par l3 a pour conséquence que la valeur de l1[3] a changé elle aussi. En revanche la valeur de l2[3] n’a pas changé.


La mutation d’un élément d’une liste ne modifie pas la longueur de cette liste. Dans la section qui suit sont présentées deux méthodes qui ont pour effet de modifier la longueur (append et pop).

Méthodes¶

Les listes possèdent onze méthodes : deux communes avec les autres structures séquentielles et neuf méthodes spécifiques.

  1. Méthodes communes index et count

    >>> l = list('timoleon')
>>> l.index('o')
3
>>> l.index('z')
...
ValueError: 'z' is not in list

  2. Méthode spécifique copy

    La méthode copy, sans paramètre, construit une copie de la liste.

    >>> l1 = list('timoleon')
>>> l2 = l1.copy()
>>> l2 == l1
True
>>> l2 is l1
False

    L’expression l1.copy() est équivalente à list(l1).

  3. Méthode spécifique append

    Remarque : Toutes les méthodes que nous présentons désormais sont des méthodes qui modifient la liste à laquelle elles s’appliquent.

    La méthode append ajoute l’élément passé en paramètre à la fin de la liste.

    >>> l = [1, 2, 3]
>>> l.append(4)
>>> l
[1, 2, 3, 4]
>>> l.append(2)
>>> l
[1, 2, 3, 4, 2]

    Note

    Ajouter un élément par concaténation ou par la méthode append ?

    S’il faut ajouter un élément en fin d’une liste existante, vaut-il mieux utiliser la concaténation ou la méthode append ?

    Observons d’abord la concaténation sur un exemple.

    On peut remarquer :

    1. que la liste (l1) d’origine n’est pas changée,
    2. qu’une copie de la liste d’origine a été construite avant de lui ajouter un élément supplémentaire.

    Observons maintenant la méthode append.

    On peut remarquer :

    1. que la liste d’origine (l1) a été modifiée
    2. et qu’en conséquence, il n’a pas fallu faire de copie.

    Alors que vaut-il mieux ? Cela dépend du contexte d’utilisation. S’il n’est pas nécessaire de conserver la liste de d’origine, l’usage de la méthode append est très certainement plus efficace, en temps et en espace, que la concaténation.

  4. Méthode spécifique pop

    La méthode pop permet de supprimer le dernier élément d’une liste et renvoie ce dernier élément.

        >>> l = [1, 2, 3, 4]
    >>> l.pop()
    4
>>> l
[1, 2, 3]

  5. Méthode spécifique reverse

    La méthode reverse ne prend pas de paramètre, et ne renvoie pas de résultat. Elle possède un effet de bord sur la liste sur laquelle elle s’applique : elle renverse l’ordre des éléments de la liste.

    >>> l=[1, 2, 3]
>>> l.reverse()
    >>> l
    [3, 2, 1]

  6. Méthode spécifique sort

    La méthode sort ne renvoie pas de résultat. Elle modifie la liste sur laquelle on l’applique. Elle range (ou trie) ses éléments dans l’ordre croissant.

    >>> l
[3, 2, 1]
>>> l.sort()
>>> l
[1, 2, 3]

    Nous étudierons deux algorithmes de tri dans un chapitre ultérieur.

Retour sur les méthodes split et join des chaînes de caractères¶

  1. Méthode split

    Cette méthode utilisée sans paramètre découpe la chaîne de caractères sur laquelle elle s’applique en une liste de chaînes obtenues en supprimant tous les caractères d’espacement : espace (' '), tabulation ('\t'), passage à la ligne ('\n').

    >>> "Timoleon est un homme\npolitique\tgrec".split()
['Timoleon', 'est', 'un', 'homme', 'politique', 'grec']
>>> "Timoleon".split()
['timoleon']

    Utilisée avec un paramètre, cette méthode découpe la chaîne en une liste de chaînes obtenues en supprimant toutes les occurrences du paramètre.

    >>> "Timoleon est un homme politique grec".split('e')
['Timol', 'on ', 'st un homm', ' politiqu', ' gr', 'c']
>>> "Timoleon est un homme politique grec".split('ol')
['Tim', 'eon est un homme p', 'itique grec']

  2. Méthode join

    D’un certain point de vue on peut considérer que la méthode join effectue le travail inverse de split : elle prend une liste de chaînes de caractères en paramètre et construit une nouvelle chaîne de caractères obtenue en intercalant la chaîne à laquelle elle s’applique toutes les chaînes de la liste.

    >>> 'e'.join(['Timol', 'on ', 'st un homm', ' politiqu', ' gr', 'c'])
'Timoleon est un homme politique grec'
>>> 'ol'.join(['Tim', 'eon est un homme p', 'itique grec'])
'Timoleon est un homme politique grec'
>>> ' '.join(['Timoleon', 'est', 'un', 'homme', 'politique', 'grec'])
'Timoleon est un homme politique grec'
>>> ''.join(['Timoleon', 'est', 'un', 'homme', 'politique', 'grec'])
'Timoleonestunhommepolitiquegrec'
>>> 'timoleon'.join([])
''
